### 2. GPT Image 2 Prompt公式（2026-05-03 更新）

**标准化模板结构**（来源：Pixverse GPT Image 2评测）：
```
[风格/媒介] + [角色锚定] + [场景/环境] + [灯光] + [构图] + [禁止项]
```

**窑滚人生角色锚定模板**：
```
JAPANESE ANCIENT-STYLE ILLUSTRATION. Clean linework, warm gray-tinted paper texture, 
cel-shading with fine hatching shadows, 45-degree light from upper-left.

QINGYIN(Qinghua): [完整外貌参数]
RUANCAI(Fencai): [完整外貌参数]
MITONG(Linglong): [完整外貌参数 - 必须强调female/silver hair/dress/cutout holes]
YAOBIAN(Color Glaze): [完整外貌参数]

SCENE: [场景描述]
[具体哪几个角色出现，什么位置，什么表情]

```

**⚠️ 角色崩溃教训（2026-05-03）**：
米通因prompt不够视觉化被GPT Image 2画成了光头和尚。原因：`drooping outer eyes + flat brows` 被理解成了僧人相。
**修法**：角色描述必须包含不可争议的视觉锚点——性别(female/girl)、发长(silver-white long hair)、裙子(dress/skirt)、体型(slim girl body)。避免只靠面部特征。

### 4. PIL云状气泡中文 (v6最终方案)

**气泡特性：**
- 云状轮廓：14个变径小圆围成，无硬边框
- 半透明 alpha=200，暖白底色 `(255, 248, 235)`
- 自适应大小：textbbox测文字→定气泡半径
- **三角形尾巴**：从气泡边缘指向说话人头部（核心！无尾巴=分不清谁说话）
- 粗字体 NotoSansCJK-Bold 18-20px，深棕(30,15,5)

**气泡定位5条铁律：**
1. **说话人名字前缀**：气泡文字开头必须加"角色名："（如"软彩：又来了..."）
2. **气泡靠近说话人**：不在中央，偏左或偏右对齐说话人
3. **三角形尾巴指向说话人**：尾巴尖指向说话人头部位置
4. **不遮脸**：气泡绝不能盖住说话人或任何角色的面部
5. **多人同框时每句话独立气泡**：每个说话人头顶一个气泡，带各自尾巴

**定位示例（1:1方形画面）：**
- 说话人在中央 → 气泡右上角(0.75,0.08)，尾巴指向(0.52,0.22)
- 说话人在右侧 → 气泡右上(0.72,0.15)，尾巴指向(0.78,0.28)
- 说话人在左侧 → 气泡左上(0.25,0.22)，尾巴指向(0.18,0.38)
- 说话人在下方 → 气泡底部(0.50,0.82)，尾巴指向(0.50,0.72)

## Ep1 实战教训（2026-05-03）

### GPT Image 2 不完全遵循 "NO text"
prompt里写"NO text on image"仍可能生成文字。原图自带文字无法通过PIL完美去除——遮盖会破坏画面。**方向**：测试GPT Image 2中文直出，省掉PIL气泡。

### 气泡迭代最佳实践
**必须加三个东西才能分清谁说话**：
1. **说话人前缀**：「青隐：四大名瓷，青花为首」
2. **三角形尾巴**：从气泡底部指向说话人头部
3. **靠近说话人**：气泡位置按角色定位（青隐左→气泡左，软彩右→气泡右）

**位置参考**：青隐(0.28, 0.12)、软彩(0.70, 0.12)、米通(0.18, 0.28)、窑变(0.55, 0.10)、无语佛(0.42, 0.08)

### 角色崩溃问题
纯文字prompt易导致角色变形：米通(银发少女)→光头和尚、窑变(渐变袍男)→黑衣女。
**解决**：T2角色注入器+参考图双保险。群像格需加CRITICAL性别标记。

### 气泡字体大小
小字号16px适合短文本（如"控制不住……"），正常对话用18px。

## 已知陷阱（2026-05-03实战补充）

### 纯文字prompt导致角色变形
只用文字描述角色外貌，GPT Image会把"玲珑瓷镂空裙害羞少女"画成光头和尚。必须用**T2角色注入器**，含CRITICAL标注。米通prompt必须标注：`CRITICAL: Female, NOT bald, NOT monk, wears dress, silver-white hair`。

### 气泡位置必须逐格迭代
第一期6格气泡迭代4轮才定稿。核心：加说话人前缀+三角形尾巴+逐格定位而不是统一坐标。

### SSH heredoc静默失败
传脚本到CORE-01用scp，不要用SSH heredoc（嵌套引号会静默失败，2026-05-03 cron部署失败根因）。
**GPT Image 2 API不支持直接输出PSD**——只有ChatGPT网页版通过Adobe Photoshop集成可以做分层编辑。API只输出PNG/JPEG/WEBP。
替代方案：角色视觉规范文档（冻结prompt模板）+ 参考图引用 + 逐格质检。

### GPT Image 2 自带文字问题（2026-05-03 实战发现）

**现象**：即使prompt写了 `NO text on image`，GPT Image 2 有时仍会自动生成文字。第5格原图自带"控制不住"四字在右上角。

**修复三步法**：
1. **先生成带气泡版本，让说话人确认哪些是多余文字**（小墨描述法：位置+内容+判断特征）
2. **精确遮盖**：只遮盖气泡文字本身，不盖背景画面
   ```python
   # 采样周边真实背景色，不要用硬编码色值
   sample = img.crop((文字左-10, 文字下+5, 文字左+10, 文字下+25))
   bg_color = average_color(sample)
   # 遮盖区域紧贴文字，不过大（通常宽48-84%，高0-14%）
   draw.rectangle([x1, y1, x2, y2], fill=bg_color)
   # 羽化底部边缘3-5%高度
   ```
3. **验证**：遮盖后画面内容不能受损（竹叶/屋檐/人物不能消失）

**铁律**：遮盖区域宁小勿大，宁可留一点残影也不要毁画面。

# 外部工具能力内化方法论

## 核心思路

> 不装别人的工具，把好东西拆解吸收成墨家军自己的技能。

## 五步法

### 第一步：全量提取
- 读透原文/仓库/文档（用web_extract/web_search覆盖）
- 列出所有能力点（每个功能一行）
- 标注：免费/收费、API支持/仅网页、通用/专用
- 关注架构文档和源码结构，不只是README

### 第二步：场景映射
- 对每个能力点问：**窑滚人生/墨家军哪个环节需要这个？**
- 不需要的直接划掉
- 需要的标注：痛点是什么、现在怎么做、用它能省什么

### 第三步：架构对比分析（新增！关键步骤）

不是"它有什么功能"而是"它是怎么做到的和我们有什么不同"。

**对比维度：**
| 维度 | 外部工具 | 墨家军 | 差距 |
|------|---------|--------|------|
| 语言/运行时 | | | |
| 核心架构 | | | |
| 关键子系统 | | | |
| 部署方式 | | | |
| 扩展机制 | | | |

**核心问题**：它解决了什么我们没解决的问题？用的什么方法？我们的架构能不能装下这个方法？

### 第四步：转化设计
- 不是"直接用"而是"重写一份墨家军版本的"
- 写明：**集成点在哪里**（具体到文件路径、注册点、hook位置）
- 写明：代码结构（模块目录、每个文件职责、行数估算）
- 写明：部署步骤（先装什么依赖、再部署什么文件、最后改什么配置）
- 写明：风险点和回滚方案

**输出模板：**
```
### 模块N：XXX
- 借鉴来源：外部工具的具体功能名
- 现状差距：我们现在怎么做/缺什么
- 集成点：Hermes tools注册点 / CORE-01 module路径
- 代码结构：
  ├── xxx.py  ← 职责（~行数）
  └── yyy.py  ← 职责（~行数）
- 部署步骤：1.装依赖 2.部署文件 3.注册工具 4.灰度测试
- 风险：低/中/高，回滚方案
```

### 第五步：优先级排序（含价值评分）
```
★★★★★ 最高优先 — 立竿见影+零风险+小代码量
★★★★☆ 高优先   — 显著提效+低风险+中等代码量
★★★☆☆ 中优先   — 锦上添花/依赖其他模块
```
按价值/代码量/风险三维排序，给出建议执行顺序。

## 输出格式

```markdown
## 架构对比

| 维度 | 外部工具 | 墨家军 | 差距 |

## 每个能力详细说明

- **借鉴来源**：外部工具的具体功能名
- **现状差距**：我们现在怎么做/缺什么

- **代码结构**：
  ```

## 实战案例

已用此方法论完成转化分析：
- DeepSeek-TUI → 墨家军：LSP诊断注入 + RLM风扇并行 + 自我更新（2026-05-03）
- MeiGen能力转化 — 7项能力，含prompt增强/脚本拆解/并行调度等（2026-05-03）
- GPT-Image2深度学习 — 7项能力，含直接中文渲染/角色一致性/PSD分层等（2026-05-03）

## 关键原则

1. **只转化真正需要的**——先做差距分析，找出真正有价值且我们缺的能力
2. **架构对比先行**——先理解"它是怎么做到的"，再设计"我们怎么做"
3. **优先做立竿见影的**——价值/代码量/风险三维排序
4. **保持墨家军架构**——不破坏现有task_queue/agent体系，增量添加
5. **写明集成点和部署步骤**——不只是在纸上谈兵，要可直接执行
6. **产出文件放 `03_产出/`**——后续可检索

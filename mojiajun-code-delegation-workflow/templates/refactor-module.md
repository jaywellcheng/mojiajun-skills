# 重构模块

## 适用场景
对现有模块进行重构——提取函数、拆分文件、优化结构、改善可读性，不改变外部行为。

## 小川填写区（3个槽位）

- 目标文件: `[FILL]`
- 改动描述: `[FILL]`
- 额外约束: `[FILL]`

## 自动生成区（以下是固定的AS prompt结构）

【任务】重构目标文件：`[FILL]`

【目标文件】[FILL]

【具体改动】
1. 提取重复代码为独立函数（≥3次重复必须提取）
2. 拆分过长的函数（>50行的函数考虑拆分为2-3个职责单一的子函数）
3. 改善命名：变量/函数名准确描述其用途
4. 添加/修正 docstring 和类型注解
5. 优化 imports 结构：标准库 → 第三方 → 本地模块，按字母序

【约束】
- 代码简洁不超100行（新函数）
- 只用stdlib除非指定
- 行为等价——重构不改变任何外部接口和返回值
- 保持原有测试兼容
- 新增函数要有 docstring
- [FILL]

【验收标准】
- python3 直接运行不报错
- 现有调用方不受影响
- 函数签名兼容（或明确标注 breaking changes）
- 代码可读性明显改善

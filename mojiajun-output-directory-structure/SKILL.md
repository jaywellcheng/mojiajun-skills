## 文件命名规范

格式: `YYYY-MM-DD_类别_内容标题.md`

示例:
- `2026-04-27_陶瓷类_爆款分析报告.md`
- `2026-04-27_热点趋势_三平台日报.md`
- `2026-04-27_卖货类_内容策略报告.md`

## 每份报告必须包含

| 模块 | 内容 | 说明 |
|------|------|------|
| 文件头 | 标题+分析日期+数据来源 | 方便追溯 |
| 核心结论 | 3-5条关键结论 | 快速阅读用 |
| 数据支撑 | 具体的数字和案例 | 要可验证 |
| 策略建议 | 对天青浅的可执行建议 | 能直接落地 |
| 更新记录 | 每次修改的日期和内容 | 迭代追踪 |

## 更新规则

1. 新增文件：当天产出按日期+类别命名
2. 迭代文件：同一主题更新在文件名加_v2/_v3
3. 过期清理：不再适用的策略打`_已废弃`后缀，不删除
4. 交叉引用：不同文件间用 `→ 详见 YYYY-MM-DD_类别_标题.md` 引用

## 数据源整理原则（来自实践 2026-04-27）

原始数据（hot_list、xhs_sample_library）可能在服务器上存放混乱，但墨家军的分析产出必须严格按结构分类存放，确保双轨学习闭环数据来源清晰。

### 处理步骤
1. **数据入库**：小墨采集→手动导入MySQL（当前阶段）
2. **分析产出**：墨家军读库分析→结果按分类存放→产出永远在本地 `/Users/jaywell/Desktop/墨家军资料库/墨家军产出文件/`
3. **数据清理**：hot_list只保留7天，每周日自动清理；样本库永久保留

### 关键原则
- 数据源可以乱（小墨采集可能IP变动导致路径不一致）
- **但产出必须干净**——墨家军输出的分析报告永远按规范分类存放
- 分析报告中必须标明数据来源和时间，方便追溯
